# Roulette Dealer Trainer

# Этап 1. Создание координатной сетки по изображению рулеточного поля

## Исходные данные

К заданию приложено изображение:

```text
rulet_field.png
```

Данное изображение является эталонным рулеточным полем.

Все координаты необходимо строить относительно этого изображения.

---

# Цель этапа

Подготовить техническую координатную систему для будущего размещения:

* фишек;
* ставок;
* подсветки номеров;
* генерации заданий.

На данном этапе игровую логику НЕ реализовывать.

---

# Что уже есть

На странице уже отображается изображение:

```text
rulet_field.png
```

Изображение находится по центру страницы на тёмном фоне.

Внешний вид изображения изменять нельзя.

---

# Что нужно сделать

Создать поверх изображения прозрачный SVG-слой.

Этот слой будет содержать координатные зоны всех номеров рулетки.

---

# Номера рулетки

Необходимо создать зоны для:

```text
0
1
2
3
...
36
```

Всего:

```text
37 зон
```

---

# Тип зон

Использовать SVG Polygon.

Не использовать div-рамки.

Не использовать обычные прямоугольники.

Причина:

На изображении присутствует перспектива.

Некоторые клетки визуально выглядят как трапеции.

Ноль имеет нестандартную форму.

---

# Формат зоны

Каждая зона должна иметь набор точек.

Пример:

```ts
{
  number: 17,
  points: [
    { x: 0, y: 0 },
    { x: 0, y: 0 },
    { x: 0, y: 0 },
    { x: 0, y: 0 }
  ]
}
```

---

# Номер 0

Номер:

```text
0
```

находится слева.

Он имеет нестандартную форму.

Для него допускается использование 6–8 точек.

Пример:

```ts
{
  number: 0,
  points: [...]
}
```

---

# Номера 1–36

Для каждого номера создать polygon-зону.

Минимум:

```text
4 точки
```

Максимум:

```text
8 точек
```

если это потребуется для более точного совпадения.

---

# Режим отладки

Добавить кнопку:

```text
Показать сетку
```

---

# При включении режима

Поверх изображения отображаются polygon-зоны.

Каждая зона должна иметь:

* жёлтую рамку;
* полупрозрачную заливку;
* номер зоны в центре.

Пример:

```text
17
```

отображается внутри polygon номера 17.

---

# При выключении режима

Видно только изображение рулетки.

Все polygon-зоны скрываются.

---

# Порядок номеров

Использовать расположение чисел как на изображении.

Верхний ряд:

```text
3   6   9   12   15   18   21   24   27   30   33   36
```

Средний ряд:

```text
2   5   8   11   14   17   20   23   26   29   32   35
```

Нижний ряд:

```text
1   4   7   10   13   16   19   22   25   28   31   34
```

---

# Точность

Polygon должен максимально точно совпадать с золотыми линиями клетки на изображении.

Особенно проверить:

* 0
* 17
* 26
* 36
* 34

Если эти номера совпадают, значит вся сетка построена правильно.

---

# Масштабирование

Если изображение изменяет размер:

* polygon-зоны должны масштабироваться вместе с ним;
* координаты не должны съезжать.

Использовать SVG Overlay.

Пример:

```tsx
<div className="roulette-wrapper">

  <img
    src="/assets/rulet_field.png"
    className="roulette-image"
  />

  <svg
    className="roulette-overlay"
    viewBox="0 0 BASE_WIDTH BASE_HEIGHT"
  >
    ...
  </svg>

</div>
```

---

# Центр зоны

Для каждой зоны необходимо вычислить центр.

В будущем центр будет использоваться для размещения фишек.

Пример:

```ts
{
  number: 17,
  centerX: 0,
  centerY: 0
}
```

---

# Что НЕ делать

На этом этапе запрещено реализовывать:

* Spin;
* генерацию случайного номера;
* ставки;
* фишки;
* выплаты;
* проверку ответа;
* звук рулетки;
* анимацию.

---

# Результат

После завершения этапа:

1. На странице отображается исходное изображение рулетки.
2. Есть кнопка:

   ```text
   Показать сетку
   ```
3. В debug-режиме отображаются polygon-зоны всех номеров.
4. Зоны совпадают с клетками на изображении.
5. Для каждой зоны определён центр.
6. Подготовлена координатная система для следующего этапа — размещения ставок и фишек.
